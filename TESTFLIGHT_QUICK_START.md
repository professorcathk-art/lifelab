# TestFlight Quick Start Guide

## 🎯 What is TestFlight?

**TestFlight is Apple's official beta testing platform** - perfect for sharing your app with testers before App Store approval!

## ✅ Why TestFlight?

- ✅ **Official Apple solution** - No hacks needed
- ✅ **Easy for testers** - Install via TestFlight app
- ✅ **Up to 10,000 testers** - Plenty for beta testing
- ✅ **Works immediately** - After uploading to App Store Connect
- ✅ **No approval needed** - TestFlight builds don't need App Store review
- ✅ **Over-the-air** - No cables or Xcode needed

## 🚀 Setup Steps (5 Minutes)

### Step 1: Upload Build (You're Doing This Anyway)

1. Archive and upload to App Store Connect
2. Same process as submitting for review
3. Build will appear in **TestFlight** tab automatically

### Step 2: Go to TestFlight Tab

1. Login to **App Store Connect**: https://appstoreconnect.apple.com
2. **My Apps** → **LifeLab**
3. Click **"TestFlight"** tab (top navigation)

### Step 3: Create Testing Group

**Option A: Internal Testing (Easiest)**

1. Click **"Internal Testing"** (left sidebar)
2. Click **"+"** to create group
3. Name it: "Beta Testers"
4. Click **"Create"**

**Option B: External Testing (For More Testers)**

1. Click **"External Testing"** (left sidebar)
2. Click **"+"** to create group
3. Name it: "Public Beta"
4. Click **"Create"**

### Step 4: Add Testers

**Method 1: Email Invites**

1. In your testing group, click **"+"** next to Testers
2. Enter email addresses (one per line)
3. Click **"Add"**
4. Testers receive email invitation

**Method 2: Public Link (Easier!)**

1. In External Testing group
2. Click **"Enable Public Link"**
3. Copy the link
4. Share link anywhere (website, social media, etc.)
5. Anyone with link can join (up to 10,000 testers)

### Step 5: Select Build

1. In your testing group
2. Click **"Select Build"** or **"+"**
3. Choose **Build 3** (your uploaded build)
4. Click **"Done"**

### Step 6: Start Testing

1. Click **"Start Testing"** or **"Submit for Review"**
   - Internal: Instant access
   - External: Needs Beta App Review (usually instant, < 24 hours)
2. Done! Testers can now install

## 📱 For Testers (What They Need to Do)

### Step 1: Install TestFlight App

1. Open **App Store** on iPhone/iPad
2. Search **"TestFlight"**
3. Install **TestFlight** app (free, by Apple)
4. Open TestFlight app

### Step 2: Accept Invitation

**If Email Invitation:**
1. Check email for invitation
2. Tap **"View in TestFlight"** or **"Start Testing"**
3. Opens TestFlight app automatically

**If Public Link:**
1. Tap the public link (from website, social media, etc.)
2. Opens TestFlight app
3. Tap **"Accept"**

### Step 3: Install Your App

1. In TestFlight app, find **"LifeLab"**
2. Tap **"Install"**
3. App installs on device
4. Tap **"Open"** to launch

### Step 4: Use App Normally

- App works exactly like App Store version
- Full functionality
- Can provide feedback via TestFlight
- Updates automatically when you upload new builds

## 🔄 Updating TestFlight Builds

**When you upload a new build:**

1. Upload new build to App Store Connect (increment build number)
2. Wait for processing (10-30 minutes)
3. Go to TestFlight → Your testing group
4. Click **"Select Build"** → Choose new build
5. Testers automatically get update notification
6. They tap **"Update"** in TestFlight app

## 📊 TestFlight Features

### Feedback Collection
- Testers can provide feedback directly in TestFlight
- Screenshots, crash logs, comments
- You receive feedback in App Store Connect

### Analytics
- See how many testers installed
- Track usage statistics
- Monitor crash reports

### Multiple Builds
- Can have multiple builds active
- Testers can choose which version to test
- Easy A/B testing

## ⚠️ Important Notes

### Internal Testing:
- ✅ Up to 100 testers
- ✅ Instant access (no review)
- ✅ Best for quick testing

### External Testing:
- ✅ Up to 10,000 testers
- ⚠️ Needs Beta App Review (usually instant, < 24 hours)
- ✅ Can create public link
- ✅ Best for public beta

### Build Expiration:
- TestFlight builds expire after 90 days
- Upload new build before expiration
- Testers get notification before expiration
